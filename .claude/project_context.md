# 项目说明：AI/LLM 文章阅读笔记系统

## 项目目的

这是一个个人知识管理系统，用于记录和整理阅读的 AI/LLM 相关技术文章、论文和博客。

## 核心概念

这是一个结构化的笔记库，通过分类体系和模板来规范文章笔记的记录方式。

## 项目结构

```
read_note/
├── articles/              # 文章笔记存放目录（所有笔记都放这里）
├── templates/             # 笔记模板
│   └── article_note_template.md
├── CATEGORIES.md          # 分类体系定义（动态更新）
├── index.md              # 总索引（所有文章的汇总列表）
└── README.md             # 项目说明
```

## 文件命名规范

**格式**: `[分类码]_序号_标题.md`

**示例**:
- `[LLM]_001_Transformer架构详解.md`
- `[ARCH]_002_注意力机制原理.md`
- `[PAPER]_003_Attention_Is_All_You_Need.md`

**规则**:
- 分类码必须是 CATEGORIES.md 中定义的有效代码
- 序号使用 3 位数字（001, 002, 003...）
- 每个分类独立计数序号
- 标题使用中文或英文，简洁明了

## 分类体系

查看 `CATEGORIES.md` 获取完整分类列表。

**一级分类（10个）**:
- `LLM` - 大语言模型
- `ARCH` - 模型架构
- `TRAIN` - 训练优化
- `AGENT` - AI Agent
- `RAG` - 检索增强
- `SAFETY` - 安全对齐
- `APP` - 应用实践
- `PAPER` - 论文研读
- `BLOG` - 技术博客
- `NEWS` - 行业动态

**二级分类**（示例）:
- `LLM.BASE` - 基础模型
- `LLM.CAP` - 能力研究
- `ARCH.TRANS` - Transformer变体
- `TRAIN.SFT` - 有监督微调

## 笔记模板

新增文章时必须使用 `templates/article_note_template.md` 模板。

**模板包含的章节**:
1. 元数据（来源、作者、日期、阅读日期、分类、标签）
2. 文章概述
3. 核心内容（主要观点、技术要点、重要发现）
4. 个人思考（有启发的点、疑问、关联）
5. 关键摘录
6. 相关资源

## 工作流程

### 添加新文章笔记的步骤：

1. **确定分类**
   - 查看 CATEGORIES.md
   - 选择合适的分类码（支持二级分类）

2. **确定序号**
   - 查看 articles/ 目录下该分类的最新序号
   - 新序号 = 最新序号 + 1

3. **创建文件**
   - 使用模板 `templates/article_note_template.md`
   - 按命名规范创建文件：`[分类码]_序号_标题.md`

4. **填写内容**
   - 完整填写元数据
   - 按模板章节填写笔记内容

5. **更新索引**
   - 在 `index.md` 中添加新文章条目
   - 更新统计信息（总文章数 +1）
   - 更新对应分类的文章列表

## 常用任务

当用户要求以下操作时，按此方式执行：

### 添加新文章笔记
1. 询问用户文章信息和分类
2. 查找该分类的最新序号
3. 创建新的笔记文件
4. 自动更新 index.md

### 查找文章
- 搜索 articles/ 目录
- 或查看 index.md 索引

### 添加新分类
1. 在 CATEGORIES.md 中添加分类定义
2. 遵循现有格式和命名规范

### 统计信息
- 查看 index.md 的统计部分
- 或搜索 articles/ 目录统计文件数

## 注意事项

1. **必须使用模板**：所有文章笔记都应基于模板创建
2. **保持索引同步**：每次添加文章后必须更新 index.md
3. **分类一致性**：使用 CATEGORIES.md 中定义的标准分类码
4. **序号唯一性**：同一分类下的序号不能重复
